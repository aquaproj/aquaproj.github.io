Only standard registry is allowed by default

#1404 #1516

If you don't set policies, aqua uses the default policy allowing only standard registries. If you set policies the default policy is ignored.

See also​

  • Tutorial - Policy
  • Reference - Policy
  • Reference - Git Repository root's policy file and policy commands

Why this change is needed​

This change makes aqua secure by default. Almost users don't need registries other than standard registry. Other registries are useful but also can be abused.

How to migrate​

info

If it is difficult to configure Policy as we describe below, you can also disable Policy by the environment variable. Please see here. Note that we don't recommend disabling Policy in terms of security.

If you use only standard registry, you don't have to do anything. Otherwise you have to set policies to allow non Standard Registries.

There are two ways to set policies.

  1. (Recommended. aqua >= v2.3.0) Add Policy files in your Git Repository's root directory and allow them by aqua policy allow command.
  2. Add Policy files and set the environment variable AQUA_POLICY_CONFIG

Git Repository's root's policy file​

aqua >= v2.3.0

Please create a Policy file in your Git Repository's root directory.

aqua policy init
vi aqua-policy.yaml
aqua policy allow

If you don't use Git, please run git init to create a directory .git. You don't have to commit files. aqua searches .git to search Git Repository's root policy file.

AQUA_POLICY_CONFIG​

If non Standard Registries are used in Global Configuration, it is useful to set AQUA_POLICY_CONFIG in .bashrc or something.

e.g. .bashrc

export AQUA_GLOBAL_CONFIG="$HOME/repos/src/github.com/aquaproj/aqua-registry/aqua-all.yaml"
export AQUA_POLICY_CONFIG="$HOME/repos/src/github.com/aquaproj/aqua-registry/aqua-policy.yaml"

If non Standard Registries are used in non Global Configuration, it would be useful to set AQUA_POLICY_CONFIG by direnv or something.

e.g. .envrc

export AQUA_POLICY_CONFIG=$PWD/aqua-policy.yaml:$AQUA_POLICY_CONFIG

You can specify multiple policy files in AQUA_POLICY_CONFIG with : and empty stings are ignored.
